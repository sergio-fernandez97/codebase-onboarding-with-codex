# Python Repository Overview

## What It Does

This is the **TheAlgorithms/Python** repository: a large educational collection of algorithm and data-structure implementations in Python.  
It is designed for learning and contribution, not as a production library API.

## Structure

- Top-level folders are topic-based (e.g., `sorts/`, `graphs/`, `data_structures/`, `dynamic_programming/`, `maths/`, `project_euler/`).
- `DIRECTORY.md` is an autogenerated index of algorithms by category.
- `scripts/` contains repo automation (e.g., regenerate `DIRECTORY.md`, validate filenames, validate Project Euler solutions).
- `docs/` and `index.md` provide Sphinx doc wiring.
- Tooling and quality gates are centralized in `pyproject.toml`, `.pre-commit-config.yaml`, and `.github/workflows/*`.

Repo scale snapshot from local scan:
- ~1,481 files total
- ~1,381 Python files
- Largest content areas include `project_euler/`, `maths/`, `data_structures/`, `graphs/`, `strings/`, and `sorts/`.

## Key Modules

- `README.md`: project purpose and contributor entry point.
- `CONTRIBUTING.md`: algorithm quality expectations (type hints, doctests, naming, behavior).
- `pyproject.toml`: dependencies and all quality tooling configuration (`ruff`, `pytest`, `mypy`, coverage, Sphinx).
- `.github/workflows/build.yml`: canonical CI test command and ignored paths in CI.
- `scripts/build_directory_md.py`: source of truth for generating `DIRECTORY.md`.
- `scripts/validate_filenames.py`: naming policy enforcement.
- `scripts/validate_solutions.py`: Project Euler correctness validation.

## Where To Start

1. Read `README.md` and `CONTRIBUTING.md` to understand conventions and acceptance criteria.
2. Use `DIRECTORY.md` to choose a domain (`sorts`, `graphs`, `dynamic_programming`, etc.).
3. Open 2-3 representative files in that domain and follow the common pattern:
   - implementation functions/classes
   - rich docstrings + doctests
   - optional `if __name__ == "__main__": doctest.testmod()`
4. Run local quality checks before changing anything:
   - `uv sync --group=test`
   - `uvx ruff check .`
   - `uv run pytest --doctest-modules`
5. For repo maintenance context, review `scripts/` and CI workflows.
