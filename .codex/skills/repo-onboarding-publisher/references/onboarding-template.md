# Onboarding Template

Use this structure unless the user explicitly asks for a different format.

## Overview

Explain:
- what the repository does
- whether it is a library, application, learning repo, or tooling repo
- the main use case or audience

## Folder Structure

Include:
- a short explanation of the top-level layout
- a compact Mermaid diagram
- a short list of the most important folders

## Key Modules

Focus on the files that define:
- project purpose
- contributor workflow
- runtime or build configuration
- indexing or automation

Prefer repo-level anchors such as:
- `README.*`
- contributor guides
- package manager manifests
- CI workflows
- scripts that generate indexes or enforce rules

## Example Explanation

Pick one beginner-friendly example and explain:
- why it is a good starting point
- the core code snippet
- the execution or algorithm flow step by step
- what it teaches about the repo style

## Learning Path

Recommend a reading order from simple to complex:
- start with repo-level docs
- then a simple category or component
- then intermediate modules
- leave advanced or specialized areas for later

Keep the path concrete. Name folders or files.
